# css编译

依赖: css-loader style-loader （有先后）

rules: {
    test:/\.css$/,
    use: [
        {
            loader: 'style-loader'，
            options: {
                insertAt: 'top', 'bottom',
                insertInto: '#myDiv',
                singleton: true,
                transform:'./transform.js'
            }
        },
        {
            loader: 'css-loader'，
            modules: true
        }
    ]
}
  • loader是从后往前执行

transform.js

module.exports=function(css) {
if(window.screen.width < 500) {
    css = css.replace('red', 'yellow')
} 
return css
}

# style-loader

核心配置：

  • nsertAt 指定区域
  • inserInto 指定dom
  • singleton 合并style标签
  • transform 插入style到页面前，用js操作

css-loader

  • module 模块化
  • minimize css4已移除
  • alias css4已移除

webpack环境配置区别 指定打包文件 # webpack --config webpack.common.js --env [env_name]

webpack原理 依赖Node环境与文件操作系统 利用node读取文件，字符串处理，再利用node写入文件

工作顺序： 读取配置文件 注册内部插件与配置插件 loader编译 按引入顺序组织模块 生成导出文件
